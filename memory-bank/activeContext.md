# 当前正在做什么 (activeContext)

> 本文档记录当前开发进度，有重大进展时请更新。

## 开发计划（来自 README）

### 第一阶段：高并发与高性能

1. **启用 Redis 分布式布隆过滤器（解决缓存穿透）** ✅
   - 废弃内存版 BloomFilter，改为基于 Redis BitMap 的分布式布隆过滤器
   - 启动时预热加载所有店铺 ID
   - 查询前校验布隆过滤器，避免无效请求打到 Redis 与 MySQL

2. **实现多级缓存 + 动态热点探测（缓解 Redis 热点 Key 瓶颈）** 🔲
   - 引入 go-cache 作为 L1 本地缓存
   - 店铺等高频接口增加热点探测逻辑
   - Hot Key 自动写入本地缓存，短 TTL（如 5 秒）

### 第二阶段：稳定性与流量治理

3. **接入 Nginx + Sentinel 双重防护** 🔲
   - Nginx 入口网关，轮询负载均衡
   - Sentinel 对秒杀接口 QPS 限流，DB 熔断降级

4. **完善定时任务 + 库存回滚** 🔲
   - Cron 扫描超时未支付订单
   - 分布式锁避免多 Pod 重复回滚
   - 关闭订单 → 恢复 MySQL 库存 → 同步 Redis 缓存

### 第三阶段：分布式集群适配

5. **改造 Redis Stream 消费者（适配多实例）** 🔲
   - 消费者名称改为 `c1-{UUID}` 等带实例标识

6. **WebSocket 实时推送 + 集群广播** 🔲
   - 下单后 WebSocket 长连接
   - Redis Pub/Sub 广播，本机连接路由推送结果
